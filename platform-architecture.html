Horizontal pipeline
Six states. One shared event bus.
Familiar names, modernized architecture. Each state is a shared platform condition — not a user screen. All three actors subscribe to the same event stream. Click any stage to expand its detail.
State diagram
Trip lifecycle as a state machine
Every trip moves through these states exactly once. Each arrow is an event emitted on the bus. Red paths are failure exits. The platform owns all transitions — no actor can skip a state.
Flowchart
Decision flow through every gate
Every diamond is a platform decision point. Green paths move forward. Red paths are exits. This is the logic the dispatching engine runs — not a user flow.
Event bus
Every state change is an event
All three actor apps and all backend services subscribe to the same canonical event stream. No actor polls. Everything reacts. These are the events that power the entire platform.
